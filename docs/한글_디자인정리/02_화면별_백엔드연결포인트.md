- `confirmedPrice`

- `harvestWindow`
- `availableKg`
- `imageUrl`

현재는 `ProductDetailViewModel.sample()` 데이터 기반입니다.

1. 상품 상세 API 응답 수신
2. 응답을 `ProductDetailViewModel`과 `HarvestSlotViewData` 구조로 매핑
3. 화면 진입 시 해당 데이터 주입

### 백엔드에게 전달할 핵심

- 예약 단위는 상품이 아니라 슬롯 기준
- 슬롯별 가격/잔여수량/상태가 필요
- 상품 공통 정보와 슬롯 정보가 함께 내려와야 함

## 4. 공통 규칙

### 공통 상태

프론트는 아래 종류 상태를 나중에 같이 붙여야 합니다.

- 로딩 상태
- 빈 데이터 상태
- 에러 상태
- 권한 없음 상태

### 권장 연결 방식

- API DTO를 바로 위젯에 넘기지 않기
- `ViewModel` 또는 `ViewData` 형태로 한번 변환 후 주입
- 디자인용 표시 포맷은 프론트에서 유지

### 백엔드 전달 메모

백엔드 담당자에게는 아래 문장으로 전달하면 됩니다.

`현재 프론트는 상품목록, 상품상세, 주문상세 기준으로 ViewModel 구조까지 정리되어 있으니, 각 화면에 필요한 API 응답을 화면별 데이터 형태로 맞춰주시면 프론트에서 매핑해서 연결하겠습니다.`
