# 현재 작업 정리

## 작업 목적

사용자 예약웹 디자인을 먼저 진행하고, 이후 백엔드 담당자가 어떤 데이터와 어떤 화면을 연결하면 되는지 쉽게 이해할 수 있도록 프론트 구조를 정리하는 것이 목적입니다.

## 지금까지 완료한 작업

### 1. 기본 앱 프로토타입 구성

- 과수원/직거래 테마 컬러 기반 웹 프로토타입 구성
- `상품목록`, `상품상세`, `주문상세` 화면 프로토타입 구현
- 상단 헤더, 네비게이션, 페이지 전환 구조 구현

### 2. MVVM 구조 정리

- `models`
- `view_models`
- `views/pages`
- `views/widgets`
- `utils`

형태로 기본 구조를 나눔

### 3. 공통 위젯 분리

다음 공통 위젯을 분리함

- 브랜드 마크
- 네비게이션 pill
- 상태 배지
- 수량 버튼
- 정보 라인
- 세로 타임라인
- 과일 일러스트 타일

### 4. 화면 전용 위젯 분리

#### 상품목록

- 소개 영역
- 필터 바
- 필터 필드
- 상품 카드

#### 상품상세

- 상단 히어로 섹션
- 수확 슬롯 카드
- 예약 요약 카드
- 상품 정보 카드

#### 주문상세

- 주문 타임라인 카드
- 배송 정보 카드
- 주문 안내 카드
- 주문 상품 요약 카드
- 반품/환불 상태 카드

### 5. 상태/샘플 데이터 정리

- 상품목록 샘플 데이터는 `ProductListViewModel`
- 상품상세 슬롯/수량 상태는 `ProductDetailViewModel`
- 주문상세 타임라인 데이터는 `OrderDetailViewModel`

형태로 정리함

### 6. 의존성 정리

- `view_model`이 `page` 타입을 직접 참조하던 구조 제거
- 화면 표시용 데이터 타입도 `view_models` 하위로 이동
- `main.dart`는 앱 셸과 페이지 전환 위주 역할만 담당하도록 정리

## 현재 상태

현재 프로젝트는 다음 상태까지 완료됨

- 디자인 프로토타입 화면 3종 존재

- 기본 상태 샘플 연결 완료
