# 남은 작업 체크리스트

이 문서는 집에서 이어서 작업할 때 우선순위 기준으로 보면 됩니다.

## 1순위: 디자인 마감

- 상품목록 실제 문구/텍스트 정리
- 상품상세 실제 문구/텍스트 정리
- 주문상세 실제 문구/텍스트 정리
- 과수원 프로젝트 톤에 맞는 폰트/간격/여백 디테일 조정
- 버튼/배지/카드 hover 상태 정리
- 반응형 폭에서 카드 간격 확인

## 2순위: 정적 샘플 화면 완성도 올리기

- 상품목록 카드에 실제 이미지 또는 이미지 영역 정책 확정
- 상품상세 상단 비주얼 교체
- 농장 정보/배송 정보/반품 정보 문구 최종 정리
- 주문상세 타임라인 텍스트 최종 정리
- 주문상세 배송/반품 카드 정보 구조 확정

## 3순위: 프론트 구조 후속 정리

- 라우팅 방식 정하기
- 공통 상수 파일 추가 정리
- 화면별 문자열 상수 분리 여부 결정
- mock repository 또는 service 폴더 도입 여부 결정

## 4순위: 백엔드 연결 준비

- 화면별 API 목록 정리
- 요청/응답 예시 JSON 문서화
- ViewModel 매핑 함수 설계
- 로딩/에러/빈 상태 UI 추가

## 5순위: 실제 연결 이후 작업

- 상품목록 API 연결
- 상품상세 API 연결
- 주문상세 API 연결
- 예외 상태 검증
- 실제 데이터 기준 UI 깨짐 점검

## 지금 바로 집에서 하기 좋은 작업 추천

### 추천 순서

1. 상품목록 문구/필터/카드 디자인 마감
2. 상품상세 문구/정보 카드/슬롯 카드 마감
3. 주문상세 타임라인/배송정보/반품 상태 문구 마감
4. API 연결표 별도 작성

## 백엔드 넘기기 전에 꼭 확인할 것

- 상품목록은 슬롯 기반 예약 구조가 보이는지
- 상품상세는 슬롯 선택 중심인지
- 주문상세는 단계형 상태와 반품 상태가 분리되어 있는지
- 샘플 문구가 실제 서비스 문구로 어느 정도 정리됐는지

## 보류해도 되는 작업

지금 당장 안 해도 되는 것

- 테스트 코드
- 실제 인증 연결
- 실제 결제 연결
- 관리자 화면

지금은 사용자 예약웹 디자인 마감과 화면별 연결 포인트 정리가 더 중요합니다.
