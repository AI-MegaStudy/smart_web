# docs/ksm 문서 정리

작성일: 2026-05-08

이 폴더는 사용자 웹 작업 과정에서 만든 기획 정리, 작업 기록, 백엔드 연결 메모를 모아둔 곳이다.

작업할 때는 아래 순서로 확인하면 된다.

## 0. 2026-05-11 기준 최신 우선 문서

현재 고객웹 시연 마감 기준은 아래 문서를 가장 먼저 확인한다.

```text
고객웹_시연마감_문서최신화_2026-05-11.md
고객웹_QA_더미데이터_생성정리_2026-05-11.md
```

최신 기준 요약:

- 고객 화면 품종 기준은 `부사 사과`, `양광 사과`이다.
- 과거 문서의 `신고/부사` 기준은 당시 작업 기록이며 현재 화면 기준이 아니다.
- 고객 화면에는 `QA`, `더미`, `API`, `DB`, `Mock`, `ML`, `테스트 코드` 같은 개발자용 표현을 노출하지 않는다.
- 반품 요청 화면은 증빙 이미지 여러 장 선택과 요청 금액 직접 입력을 지원한다.
- 이용약관, 개인정보처리방침은 고객센터 안내가 아니라 본문 모달로 표시한다.

## 0.1 2026-05-10 기준 우선 문서

현재 실제 API 연결과 QA 상태를 볼 때는 아래 문서를 먼저 확인한다.

```text
사용자웹_최종QA_및_백엔드전달_2026-05-10.md
백엔드_임시수정_전달_2026-05-10.md
앞으로_해야할_작업.md
```

현재 상태 요약:

- 상품, 예약, 주문, 결제, 주문 목록, 주문 상세, 마이페이지, 회원정보 수정, 배송지 API 연결은 진행 완료.
- 이메일 인증은 DB에서 `attempt_count`, `updated_at`, `account_id NULL 허용`, `verification_code VARCHAR(255)` 보정 후 메일 발송, 코드 검증, 회원가입, 로그인을 확인했다.
- 백엔드 환경변수는 프로젝트 루트 `.env`를 기준으로 관리한다.
- 로그인/회원가입 기본 테스트 값과 고객 화면의 개발자용, 시연용 문구를 제거했다.
- 배송/반품 최종 성공 검증은 농가/관리자 앱에서 배송 상태 변경 흐름이 연결된 뒤 다시 확인.
- 고객 화면에는 `Mock`, `더미`, `API`, `DB`, `테스트 인증 코드` 같은 개발자용 표현을 노출하지 않는다.

## 1. 현재 기준으로 먼저 볼 문서

### 팀장 컨펌 체크리스트

```text
팀장컨펌_작업체크리스트_2026-05-07.md
```

팀장 피드백을 기준으로 남은 작업과 완료된 작업을 확인하는 문서다.

현재 사용자 웹 작업을 이어갈 때 가장 먼저 확인한다.

### 앞으로 해야 할 작업

```text
앞으로_해야할_작업.md
```

남은 작업, API 연결 전 준비, 발표 전 확인 항목을 정리한 문서다.

작업 우선순위를 잡을 때 확인한다.

### 사용자 웹 추가 기능 제안

```text
사용자웹_추가기능_제안.md
```

기획서 범위 안에서 추가해도 좋은 사용자 편의 기능을 정리한 문서다.

새 기능을 추가할지 판단할 때 확인한다.

### 마이페이지, 배송지, 회원 정보 API 연결 정리

```text
마이페이지_배송지_회원정보_API연결정리.md
```

마이페이지, 배송지 관리, 회원 정보 화면을 실제 백엔드와 연결할 때 필요한 API와 필드를 정리한 문서다.

백엔드 담당자에게 전달할 때 확인한다.

### 백엔드 전달 요약

```text
백엔드_전달요약_2026-05-08.md
```

추천 상품 3개, 배송지 필드, 주문 상태 문구, 배송 정보, 반품, 카드 결제 승인 API 기준을 한 번에 전달하기 위한 요약 문서다.

팀원에게 빠르게 공유할 때 이 문서를 우선 사용한다.

### 최종 QA 및 백엔드 전달 정리

```text
사용자웹_최종QA_및_백엔드전달_2026-05-10.md
```

실제 API 연결 후 사용자가 확인한 QA 항목, 이메일 인증 최신 상태, 배송/반품 추후 검증 항목, 추가한 API 목록을 정리한 최신 문서다.

### 백엔드 임시 수정 전달

```text
백엔드_임시수정_전달_2026-05-10.md
```

백엔드 폴더에서 임시로 수정하거나 추가한 파일, DB 스키마 불일치와 최신 보정 상태, 회원정보 수정 API, 배송지 API 내용을 백엔드 담당자에게 전달하기 위한 문서다.

### 이메일 인증 문서

```text
이메일_인증_최신정리_2026-05-10.md
email/email_verification_signup_test.md
```

팀장 전달 기준으로 이메일 인증 DB 보정 내용과 회원가입 테스트 절차를 정리한 최신 문서다.

핵심 기준:

- `email_verifications.attempt_count`는 실패 횟수 제한에 사용한다.
- `email_verifications.updated_at`은 백엔드 모델의 `TimestampMixin` 기준으로 매핑한다.
- 가입 전 인증을 위해 `email_verifications.account_id`는 `NULL`을 허용한다.
- `verification_code`는 해시 저장을 위해 `VARCHAR(255)`를 사용한다.
- 실제 백엔드 설정은 `smart_web/.env`를 기준으로 한다.

### 커밋/푸시 요약

```text
커밋_푸시_요약_2026-05-10.md
```

이번 변경의 커밋 메시지 후보, 주요 변경 파일, 검증 결과, 푸시 전 확인 사항을 정리한 문서다.

## 2. 작업 기록 문서

아래 문서들은 날짜별 작업 기록이다.

현재 기준 문서와 내용이 일부 겹칠 수 있지만, 언제 어떤 작업을 했는지 확인하기 위해 기록용으로 유지한다.

```text
2026-05-06_최근_작업_진행_기록.md
2026-05-07_인증_예약안정성_UI보강_작업기록.md
2026-05-08_팀장컨펌_반영_작업기록.md
작업_진행_기록.md
```

정리 기준:

- 날짜별 파일은 상세 작업 히스토리로 유지한다.
- `작업_진행_기록.md`는 전체 진행 요약으로 유지한다.
- 새 날짜 작업이 길어지면 날짜별 작업기록을 만들고, 중요한 결론만 현재 기준 문서에 반영한다.

## 3. 기획 및 참고 문서

### 사용자 웹 필요한 정보 정리

```text
사용자웹_필요정보_정리.md
```

사용자 웹의 전체 목표, 화면 흐름, API 연결 지점, 화면 문구 원칙을 정리한 문서다.

새 화면을 만들거나 흐름을 다시 확인할 때 참고한다.

### DB 수정 없는 개선 우선순위

```text
사용자웹_DB수정없는_개선우선순위.md
```

DB 변경 없이 프론트에서 먼저 개선할 수 있는 홈 화면 구성, 애니메이션, 푸터, FAQ 등을 정리한 문서다.

홈 화면 개선 방향을 잡을 때 참고한다.

## 4. 중복 정리 기준

- 같은 날짜의 세부 작업 문서가 이미 큰 작업기록 안에 포함되어 있으면 별도 문서를 만들지 않는다.
- 날짜별 기록 문서는 삭제하지 않고 기록용으로 유지한다.
- 현재 작업 판단은 `앞으로_해야할_작업.md`, `사용자웹_최종QA_및_백엔드전달_2026-05-10.md`, `백엔드_임시수정_전달_2026-05-10.md`를 우선한다.
- 백엔드 전달 내용은 기능 문서에 흩어져 있더라도 최종 전달 전 `사용자웹_최종QA_및_백엔드전달_2026-05-10.md` 또는 `백엔드_임시수정_전달_2026-05-10.md`에 모은다.
