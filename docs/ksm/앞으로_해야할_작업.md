# 사용자 웹 앞으로 해야 할 작업

최신화: 2026-05-06

- 주문서 작성 화면
- 결제 화면
- 주문 완료 화면
- 내 주문 화면
- 주문 상세 화면
- 반품 신청 화면
- 반품 요청 완료 화면
- 로그인 화면
- 회원가입 화면
- 이메일 인증 화면
- 마이페이지 화면

## 2. 최근 완료된 보강 작업

2026-05-06 기준으로 아래 작업을 완료했다.

- 상단바에 `마이페이지` 메뉴를 항상 보이도록 추가
- 비로그인 상태에서 마이페이지 접근 시 로그인 필요 안내 처리
- 모바일 상품 카드 높이와 내부 여백 축소
- 홈 상품 카드 클릭 시 해당 상품 상세 페이지로 바로 이동
- 예약/주문/반품 흐름 페이지 상단에 현재 진행 상태 배지 추가
- 마이페이지 상단 요약 카드 추가
- 마이페이지 하단의 중복 기본 배송지 카드 제거
- 배송지 관리/추가/수정 화면 추가
- 배송지 추가/수정 화면 필수 입력 검증 추가
- 배송 요청사항 `직접 입력` 옵션 추가
- 회원 정보 보기/수정 화면 추가
- 이메일 인증 상태를 기본 정보 안의 `인증 완료` 배지로 표시
- 마이페이지, 배송지, 회원 정보 API 연결 지점 문서화
- 결제 화면을 수확 예약 서비스에 맞는 `결제 전 확인` 구조로 재구성
- 주문 완료/주문 상세 화면을 결제 후 `농가 확인 중` 흐름에 맞게 정리

## 3. 앞으로 우선 진행할 작업

| 우선순위 | 작업 | 목적 |
|---|---|---|
| 1 | 전체 모바일 QA | 실제 모바일 크기에서 overflow, 버튼 밀림, 여백 문제 확인 |
| 2 | 더미 데이터 최종 통일 | 상품명, 주문번호, 날짜, 금액, 상태값을 화면 전체에서 일관되게 유지 |
| 3 | 빈 상태/오류 상태 보강 | 빈 예약함, 빈 주문 목록, 로딩 실패 화면 완성 |
| 4 | 백엔드 API 연결 준비 | 담당자에게 요청/응답 필드 전달 |
| 5 | 실제 API 연동 | 더미 Repository를 실제 API Repository로 교체 |
| 6 | 로그인 세션 연결 | Mock 로그인 상태를 실제 토큰 기반 상태로 교체 |
| 7 | 카카오 주소 API 연결 | 배송지 추가/수정 화면의 주소 찾기 기능 연결 |
| 8 | 최종 발표/시연 QA | 데스크톱/모바일 시연 흐름 점검 |

## 4. 더미 데이터 최종 통일

화면별 더미 데이터가 아직 완전히 같은 기준으로 관리되는 것은 아니다.

정리할 항목:

- 상품명
- 농장명
- 수확 예정 날짜
- 패키지 단위
- 잔여 수량
- 주문번호
- 주문 상태
- 배송 상태
- 결제 금액
- 반품 가능 주문 상태
- 회원 이름, 이메일, 휴대폰번호
- 기본 배송지

권장 기준:

```text
대표 주문번호: ORD-20261012-008
대표 상품: 후지 사과
대표 농장: 청송 햇살농원
대표 수확 예정: 10.12-10.18
대표 회원: 홍길동
대표 이메일: customer@test.com
대표 연락처: 010-1111-2222
대표 배송지: 서울시 강남구 테헤란로 123
대표 택배사: CJ대한통운
대표 송장번호: 5891-1202-4810
결제 직후 상태: 농가 확인 중
```

## 5. 빈 상태/오류 상태 보강

현재는 대부분 더미 데이터가 있는 상태를 기준으로 화면이 구성되어 있다.

추가하면 좋은 상태:

- 예약함이 비어 있을 때
- 주문 목록이 비어 있을 때
- 배송지 목록이 비어 있을 때
- 상품 목록이 비어 있을 때
- 상품 이미지 로딩 실패
- API 로딩 실패
- 네트워크 오류

고객 화면에는 API 경로, DB명, 내부 상태 코드가 노출되지 않도록 한다.

## 6. 백엔드 연결 준비

백엔드 담당자에게 전달해야 할 주요 연결 지점은 아래와 같다.

### 인증/회원

- `POST /auth/customers/signup`
- `POST /auth/email/verify`
- `POST /auth/email/resend`
- `POST /auth/login`
- `GET /me`
- `PATCH /me`

### 마이페이지

- `GET /me`

필요 데이터:

- 이름
- 이메일
- 휴대폰번호
- 이메일 인증 여부
- 진행 중 주문 수
- 최근 주문 상태
- 기본 배송지 요약

자세한 내용은 `docs/ksm/마이페이지_배송지_회원정보_API연결정리.md`를 기준으로 한다.

### 배송지

- `GET /me/addresses`
- `POST /me/addresses`
- `PATCH /me/addresses/{address_id}`
- `PATCH /me/addresses/{address_id}/default`

### 상품

- `GET /products?featured=true`
- `GET /products`
- `GET /products/{product_id}`
- `GET /products/{product_id}/slots`

### 예약

- `POST /reservations/preview`
- `POST /reservations`

### 주문/결제

- `POST /orders/from-reservation`
- `POST /payments/mock-approve`
- `GET /me/orders`
- `GET /me/orders/{order_id}`
- `GET /me/orders/{order_id}/shipment`

### 반품

- `POST /returns`

## 7. 실제 API 연동 전 준비

실제 API 연동 전 아래 구조를 정리하면 좋다.

- 더미 Repository와 실제 API Repository 분리
- 공통 API 클라이언트 정리
- 인증 토큰 저장 위치 정리
- API 오류 메시지 고객용 문구로 변환
- 로딩/성공/실패 상태 ViewModel 기준 통일
- 라우트 arguments 타입 정리

## 8. 발표 또는 시연 전 체크리스트

- 데스크톱 첫 화면 확인
- 모바일 첫 화면 확인
- 홈 상품 클릭 시 상품 상세 이동 확인
- 상품 상세에서 예약함 담기 확인
- 예약함 수량 변경/선택 삭제 확인
- 예약 확인 → 주문서 작성 → 결제 → 주문 완료 흐름 확인
- 주문 상세 상태 타임라인 확인
- 반품 신청 → 반품 요청 완료 흐름 확인
- 로그인, 회원가입, 이메일 인증 이동 확인
- 마이페이지 진입 확인
- 배송지 관리/추가/수정 확인
- 회원 정보 보기/수정 확인
- 모바일 overflow 경고 확인
- `dart analyze` 통과 확인
