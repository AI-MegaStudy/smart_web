## 3. 배송 정보 입력 보강

주문서 작성 화면의 배송 정보 입력 영역을 체크리스트 기준에 맞게 보강했다.

완료 내용:

- 주문서 작성 화면에 `배송지 선택` 카드 영역을 추가했다.
- 더미 배송지로 `내집`, `부모님댁`을 구성했다.
- 배송지 카드를 선택하면 받는 분, 연락처, 주소, 배송 메모가 자동으로 바뀐다.
- 선택한 배송지에 별명이 있으면 주문서 요약의 배송 정보에 `배송지: 별명`을 표시한다.
- 별명이 비어 있으면 해당 줄은 표시하지 않는다.
- 배송지 관리 카드의 레이아웃을 정리했다.
  - 첫 줄: 별명, 기본 배송지, 최근 사용 배지
  - 다음 줄: 받는 분
  - 다음 줄: 주소
  - 다음 줄: 연락처
- 배송지 추가/수정 화면에 `배송지 별명 (선택)` 입력 필드를 추가했다.
- 배송지 별명은 선택 입력으로 처리한다.

백엔드 전달 필요:

- 배송지 목록 조회 API가 필요하다.
- 주문서 작성 화면에서 선택할 배송지 목록 응답에 아래 필드가 있으면 좋다.

```text
address_id
address_label
receiver_name
receiver_phone
zip_code
address
detail_address
delivery_memo
is_default
is_recent
```

## 4. 결제 수단 보강

결제 화면에 더미 결제 수단과 준비중 안내를 추가했다.

완료 내용:

- 결제 수단 드롭다운을 추가했다.
- 선택지는 아래와 같다.
  - 카드 간편결제
  - 무통장 입금
  - 카카오페이
  - 네이버페이
- 기본 선택은 `카드 간편결제`다.
- `카드 간편결제` 선택 후 확인 체크를 하면 주문 완료 화면으로 이동한다.
- 다른 결제 수단 선택 후 결제하면 준비중 팝업을 표시한다.

준비중 팝업 문구:

```text
해당 결제 수단은 준비 중입니다. 현재는 카드 간편결제로 시연할 수 있습니다.
```

## 5. 예약 확인 요약 카드 보강

예약 확인 페이지의 오른쪽 요약 카드 문구와 안내 강조를 정리했다.

완료 내용:

- 중복 느낌이 있던 `예약 요약`, `예약 내용 요약` 표현을 정리했다.
- 카드 제목을 `예약 상품 확인`으로 변경했다.
- 카드 배경은 흰색으로 유지했다.
- 안내 문구 2개를 역할별 박스로 분리했다.

안내 박스:

- `수확 일정 안내`
  - 기상과 생육 상황에 따라 일정이 조정될 수 있음을 안내한다.
  - 크림/노랑 톤으로 표시한다.
- `다음 단계 안내`
  - 배송 정보와 결제 전 금액을 다시 확인한다는 흐름을 안내한다.
  - 연한 초록 톤으로 표시한다.

## 7. 홈 수확 슬롯 노출 정책

홈의 `이번 주 오픈 수확 슬롯` 노출 방식은 이미 대표 3개 제한으로 구현되어 있음을 확인했다.

현재 프론트 구현:

```dart
fetchFeaturedProducts() => _products.take(3)
```

백엔드 전달 추천:

```text
GET /products?featured=true&limit=3
```

상품 목록 페이지는 별도 전체 목록을 사용한다.

```text
GET /products
```

품종 필터는 상품 목록 페이지에 이미 있는 흐름을 유지하고, 홈에는 필터를 추가하지 않는 방향이 적절하다.

### 메인 화면 섹션 카드화

- 품종 소개 영역을 하나의 흰색 카드 패널로 정리했다.
- 품종별 카드가 모바일에서 세로로 과하게 늘어나거나 텍스트가 잘리는 문제를 줄이기 위해 `GridView` 대신 `Wrap` 기반 배치로 정리했다.
- 보관 안내 영역을 별도 카드 패널로 구성하고 아이콘, 테두리, 그림자를 추가해 다른 설명 영역과 구분되도록 했다.
- 즐기는 방법, 예약 흐름, 자주 묻는 질문 영역도 공통 카드 패널 구조로 맞췄다.
- 모바일에서 카드 간 여백이 너무 넓어 보이지 않도록 내부 카드 높이와 여백을 조정했다.

### 메인 화면 빠른 이동 UI

- 긴 홈 화면에서 사용자가 원하는 구역으로 바로 이동할 수 있도록 빠른 이동 메뉴를 추가했다.
- 이동 항목은 아래와 같다.

```text
위로
상품
품종
보관
흐름
FAQ
```

- 각 버튼은 해당 구역으로 부드럽게 스크롤 이동한다.
- 모바일에서는 하단 중앙에 작은 아이콘 메뉴로 표시한다.
- 데스크톱에서는 오른쪽 중간 위치에 세로 메뉴로 표시한다.
- 모바일 하단 메뉴가 푸터 내용을 가리지 않도록 푸터 하단 여백을 추가했다.

### 회원가입 실시간 검증

- 회원가입 버튼 클릭 후 팝업으로만 안내하던 방식 대신, 입력 중 바로 상태를 확인할 수 있는 검증 문구를 추가했다.
- 검증 항목은 아래와 같다.

```text
이메일 형식
비밀번호 8자 이상
이름 2자 이상
전화번호 숫자 10~11자리
```

- 조건을 만족하면 초록색 체크 안내, 부족하면 빨간색 안내로 표시한다.
- 모든 조건을 만족하기 전에는 `가입하고 인증` 버튼이 비활성화된다.
