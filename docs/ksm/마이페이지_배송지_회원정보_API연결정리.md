# 마이페이지, 배송지, 회원 정보 API 연결 정리

## 1. 문서 목적

현재 사용자 웹에서는 마이페이지, 배송지 관리, 회원 정보 화면을 프론트 더미 데이터로 구현했다.

이 문서는 추후 백엔드 담당자에게 전달할 API 연결 지점과 필요한 응답 필드를 정리하기 위한 문서다.

## 2. 현재 프론트 구현 범위

현재 구현된 화면은 다음이다.

- 마이페이지
- 배송지 관리
- 배송지 추가
- 배송지 수정
- 회원 정보 보기
- 회원 정보 수정

현재는 실제 API 호출 없이 더미 데이터와 안내 팝업으로 동작한다.

## 3. 마이페이지 연결 지점

### 필요한 API

```text
GET /api/v1/me
```

### 화면에서 필요한 정보

| 화면 표시 | 필요 필드 예시 |
|---|---|
| 이름 | `name` |
| 이메일 | `email` |
| 휴대폰번호 | `phone` |
| 이메일 인증 여부 | `email_verified` |
| 진행 중 주문 수 | `active_order_count` |
| 최근 주문 상태 | `recent_order_status` |
| 기본 배송지 요약 | `default_address_summary` |

### 프론트 표시 기준

- `email_verified = true`이면 `인증 완료` 배지를 표시한다.
- `email_verified = false`이면 추후 `인증 필요` 또는 `미인증` 배지를 표시할 수 있다.
- 진행 중 주문 수와 최근 주문 상태는 마이페이지 상단 요약 카드에 표시한다.

## 4. 배송지 관리 연결 지점

### 배송지 목록 조회

```text
GET /api/v1/me/addresses
```

### 응답 필드 예시

| 필드 | 설명 |
|---|---|
| `address_id` | 배송지 ID |
| `address_label` | 배송지 별명. 예: 내집, 부모님댁, 회사. 선택값 |
| `receiver_name` | 받는 분 이름 |
| `phone` | 휴대폰번호 |
| `zip_code` | 우편번호 |
| `address` | 기본 주소 |
| `detail_address` | 상세 주소 |
| `delivery_request` | 배송 요청사항 |
| `is_default` | 기본 배송지 여부 |
| `is_recent` | 최근 사용 여부 |

### 프론트 표시 기준

- `is_default = true`이면 `기본 배송지` 배지를 표시한다.
- `is_recent = true`이면 `최근 사용` 배지를 표시한다.
- `address_label`이 있으면 배송지 카드와 주문서 작성 화면에 별명을 표시한다.
- `address_label`이 비어 있으면 별명 줄은 표시하지 않는다.
- 기본 배송지로 선택된 주소는 주문서 작성 화면의 초기 배송지로 사용할 수 있다.

## 5. 배송지 추가 연결 지점

### 필요한 API

```text
POST /api/v1/me/addresses
```

### 요청 필드 예시

```json
{
  "address_label": "내집",
  "receiver_name": "홍길동",
  "phone": "010-1111-2222",
  "zip_code": "06236",
  "address": "서울시 강남구 테헤란로 123",
  "detail_address": "101동 1203호",
  "delivery_request": "문 앞에 놓아주세요",
  "is_default": true
}
```

### 현재 프론트 검증

- 이름 필수
- 배송지 별명 선택
- 휴대폰번호 필수
- 주소 필수
- 상세주소 필수
- 배송 요청사항에서 `직접 입력`을 선택한 경우 직접 입력값 필수

휴대폰번호 형식 검증은 아직 약하게 두었다. 필요 시 백엔드 정책에 맞춰 강화한다.

## 6. 배송지 수정 연결 지점

### 필요한 API

```text
PATCH /api/v1/me/addresses/{address_id}
```

### 요청 필드 예시

배송지 추가와 동일한 필드를 사용한다.

```json
{
  "address_label": "내집",
  "receiver_name": "홍길동",
  "phone": "010-1111-2222",
  "zip_code": "06236",
  "address": "서울시 강남구 테헤란로 123",
  "detail_address": "101동 1203호",
  "delivery_request": "배송 전 연락해주세요",
  "is_default": true
}
```

## 7. 기본 배송지 설정 연결 지점

기본 배송지 설정을 별도 API로 둘 경우 아래처럼 분리할 수 있다.

```text
PATCH /api/v1/me/addresses/{address_id}/default
```

또는 배송지 추가/수정 요청의 `is_default` 필드로 함께 처리할 수 있다.

프론트 입장에서는 둘 중 하나만 명확하면 된다.

## 8. 카카오 주소 API 연결 지점

현재 `주소 찾기` 버튼은 안내 팝업만 표시한다.

추후 연결 시 필요한 작업:

- 카카오 주소 검색 UI 또는 Web SDK 연결
- 우편번호 자동 입력
- 기본 주소 자동 입력
- 상세주소는 사용자가 직접 입력

프론트에서 필요한 최종 값:

| 화면 필드 | 저장 필드 |
|---|---|
| 우편번호 | `zip_code` |
| 주소 | `address` |
| 상세주소 | `detail_address` |

## 9. 회원 정보 조회 연결 지점

### 필요한 API

```text
GET /api/v1/me
```

### 화면 표시 필드

| 화면 표시 | 필요 필드 예시 |
|---|---|
| 이름 | `name` |
| 이메일 | `email` |
| 이메일 인증 여부 | `email_verified` |
| 휴대폰번호 | `phone` |

## 10. 회원 정보 수정 연결 지점

### 필요한 API

```text
PATCH /api/v1/me
```

### 요청 필드 예시

```json
{
  "name": "홍길동",
  "email": "customer@test.com",
  "phone": "010-1111-2222"
}
```

현재 프론트 검증:

- 이름 필수
- 이메일 필수
- 휴대폰번호 필수

이메일 형식 검증은 추후 정책에 맞춰 강화할 수 있다.

## 11. 이메일 인증 연결 지점

회원 정보 수정 화면에는 이메일 옆에 `인증하기` 버튼이 있다.

현재는 안내 팝업만 표시한다.

추후 필요한 API:

```text
POST /api/v1/auth/email/resend
POST /api/v1/auth/email/verify
```

### UX 기준

- 이메일 인증은 선택 사항으로 둘 수 있다.
- 이메일을 변경해도 저장은 가능하다.
- 인증 완료 전에는 회원 정보 화면에서 `인증 필요` 또는 `미인증` 배지를 표시할 수 있다.
- 인증 완료 후에는 `인증 완료` 배지를 표시한다.

## 12. 주문서 작성 화면과의 연결

배송지 관리 기능은 주문서 작성 화면과 연결된다.

주문서 작성 화면 진입 시:

```text
GET /api/v1/me/addresses
```

를 통해 기본 배송지를 불러오고, `is_default = true`인 주소를 초기값으로 표시하면 된다.

현재 프론트 더미 화면에서는 주문서 작성 화면에서 배송지 카드 목록을 보여주고, 사용자가 배송지를 선택하면 아래 값이 자동으로 채워진다.

- 받는 분
- 연락처
- 주소
- 배송 요청사항

선택한 배송지에 별명이 있으면 주문서 요약에 `배송지: 별명` 형태로 표시한다. 별명이 없으면 해당 줄은 숨긴다.

배송지를 추가하거나 수정한 뒤 주문서 작성 화면으로 돌아왔을 때는 최신 기본 배송지를 다시 반영하는 구조가 필요하다.

## 13. 백엔드 담당자에게 전달할 핵심 요약

- 마이페이지 상단 요약을 위해 `GET /me` 응답에 주문 요약 정보가 필요하다.
- 배송지 목록/추가/수정/기본 배송지 설정 API가 필요하다.
- 배송지 별명은 선택값으로 받고, 응답에 포함되면 프론트에서 표시한다.
- 주소 검색은 카카오 주소 API 연동 예정이며, 저장은 우편번호/주소/상세주소로 분리하면 된다.
- 회원 정보 수정은 `PATCH /me` 기준으로 연결하면 된다.
- 이메일 인증은 필수가 아니어도 되며, 인증 여부만 화면에서 배지로 표시할 수 있으면 된다.
