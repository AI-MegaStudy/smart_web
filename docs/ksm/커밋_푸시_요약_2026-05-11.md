
## 주요 변경 요약

### 상품/농장

- 상품 상세에 `products.product_description` 연결
- 상품 응답에 농장 상세 필드 추가
  - `farm_address`
  - `farm_description`
  - `delivery_policy`
  - `return_policy`
- 농장 이미지는 `farm_image_url`이 있을 때만 표시
- 농장 이미지가 없거나 로드 실패하면 대체 박스 없이 숨김
- 고객 화면 기준 품종을 `부사 사과`, `양광 사과`로 정리
- `Shingo`, `신고`, `yanggwang`, `yangkwang` 응답은 고객 화면에서 `양광`으로 보정

### 홈

- 홈 `대표 상품` 버튼을 예약 가능한 사과 중 잔여 수량이 가장 많은 상품의 상세 화면으로 연결
- 이번 달 오픈 수확 슬롯은 현재 월 기준 최대 3개만 표시

### 예약함/결제

- 로컬 예약함의 시연용 기본 상품 제거
- 결제 완료 후 로컬 예약함 비우기
- 결제 정보에 `payment_method`, `approved_at` 연결
- 결제 일시가 있으면 주문 상세 결제 정보에 표시

### 주문 상세

- 주문 상세에 아래 필드 연결
  - `receiver_phone`
  - `delivery_memo`
  - 서버 확정 `total_amount`
- 주문 금액은 프론트 합산보다 서버 확정 금액을 우선 사용
- 배송 Repository에는 `shipped_package_count`, `shipped_kg`를 연결했지만 화면 중복을 피하기 위해 주문 상세에는 표시하지 않음

### 반품

- 반품 내역에 아래 처리 결과 필드 연결
  - `approved_amount`
  - `decision_reason`
  - `decided_at`
- 값이 있을 때만 고객 화면에 표시
- 반품 증빙 이미지 임시 경로에서 `mock` 표현 제거

### QA/문구 정리

- `QA 수확 슬롯 308` 같은 슬롯 안내 문구는 고객 화면에서 숨김
- 고객 화면 검색 기준 아래 표현 직접 노출 없음 확인
  - `Harvest Demo Farm`
  - `customer@test.com`
  - `Mock 결제`
  - `database integrity error`
  - `/mock/returns`
  - `더미`
  - `DB`
  - `API`

## DB 조회 결과

DB는 수정하지 않고 후보만 조회했다.

남은 데이터 후보:

- `products.image_url` 비어 있는 상품: `product_id 1, 2, 5, 6`
- `product_id 6`: `fruit_type = 사과`, 필요 시 `APPLE`로 통일 후보
- `harvest_slots.slot_id 4 ~ 27`: `customer_notice`에 `QA 수확 슬롯 ...` 문구 남아 있음
- `slot_id 2`: 닫힌 테스트 슬롯 문구 남아 있음

현재 화면에서는 QA 슬롯 문구는 숨김 처리되어 있다.
