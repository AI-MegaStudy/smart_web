## 2. 주요 변경 요약

### 백엔드

- `email_verifications.attempt_count`를 다시 모델과 검증 로직에 연결했다.
- `EmailVerification` 모델이 `TimestampMixin`을 다시 사용하도록 복구했다.
- 가입 전 이메일 인증 후 회원가입이 완료되면 인증 레코드의 `account_id`를 새 계정과 연결하도록 했다.
- 백엔드 환경변수 기준을 프로젝트 루트 `.env`로 정리했다.

### 프론트

- 로그인/회원가입 화면의 테스트 이메일, 비밀번호, 이름, 전화번호 기본값을 제거했다.
- 회원가입 검증 문구는 사용자가 입력한 필드에만 표시되도록 조정했다.
- 결제 화면의 시연용 문구와 `결제 승인` 표현을 고객용 문구로 바꿨다.
- 회원정보 화면의 `팀장 확인`, `연결 예정`, `인증 정책 확인` 문구를 고객센터 문의 안내로 바꿨다.
- 홈 푸터의 `백엔드 연결 후` 문구를 고객센터 문의 안내로 바꿨다.
- 예약/주문 실패 문구에서 `생성 실패` 표현을 고객용 안내로 바꿨다.
- 알 수 없는 서버 오류가 그대로 노출되지 않도록 기본 고객용 오류 문구를 적용했다.

### 문서

- `AGENTS.md`에 이메일 인증 DB 보정 완료, 루트 `.env` 기준, 이메일 인증 연결 기준을 반영했다.
- `docs/ksm/email` 문서를 확인하고 상위 요약 문서로 `이메일_인증_최신정리_2026-05-10.md`를 추가했다.
- `README.md`, `앞으로_해야할_작업.md`, `작업_진행_기록.md`, `사용자웹_최종QA_및_백엔드전달_2026-05-10.md` 등 최신 문서를 갱신했다.

## 5. 푸시 전 체크

```powershell
git status --short
python -m py_compile backend\app\models\account.py backend\app\services\email_verification_service.py backend\app\services\auth_service.py backend\app\core\config.py
flutter analyze
```

`flutter analyze`가 오래 걸리면 아래 순서로 정리 후 재시도한다.

```powershell
Get-Process | Where-Object { $_.ProcessName -match 'dart|flutter|dartvm' } | Stop-Process -Force
flutter clean
flutter pub get
flutter analyze
```

## 6. 커밋 대상 주의

- 루트 `.env`와 `backend/.env`는 민감정보가 들어갈 수 있으므로 커밋 대상에 포함하지 않는다.
- `docs/ksm/email/`은 팀장 전달 이메일 인증 문서이므로 이번 문서 최신화와 함께 포함 가능하다.
- 고객 화면 문구 변경은 Flutter 화면 파일과 ViewModel 변경을 함께 커밋한다.
