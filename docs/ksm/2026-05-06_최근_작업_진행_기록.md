# 2026-05-06 사용자 웹 최근 작업 진행 기록

## 1. 작업 목적

팀장 피드백과 사용자 웹 완성도 보강을 기준으로 홈, 상품 카드, 예약 흐름, 마이페이지, 배송지, 회원 정보 화면을 정리했다.

이번 작업은 백엔드 API를 직접 연결하지 않고, 프론트 더미 화면과 화면 흐름을 실제 쇼핑몰/예약 서비스처럼 보이게 만드는 데 초점을 맞췄다.

## 2. 상단바와 마이페이지 접근

- 홈 상단바에 `마이페이지` 메뉴를 항상 보이도록 추가했다.
- 비로그인 상태에서 `마이페이지`를 누르면 로그인 필요 안내 후 로그인 화면으로 이동한다.
- 로그인 상태에서는 상단바의 `회원가입`, `로그인` 메뉴를 숨기고 `마이페이지`와 예약함 중심으로 보이게 정리했다.
- 로그아웃은 상단바가 아니라 마이페이지 내부에서 처리하도록 유지했다.

## 3. 모바일 반응형 개선

- 상품 목록 모바일 카드 높이를 줄였다.
- 홈의 추천 상품 카드도 상품 목록과 같은 밀도 기준으로 조정했다.
- 모바일에서 상품 이미지 높이, 카드 내부 여백, 칩 간격을 줄였다.
- 홈 hero 영역의 모바일 높이와 세로 여백을 줄였다.
- 홈 섹션 제목과 상품 목록 상단 설명 영역의 모바일 여백을 줄였다.

## 4. 홈 상품 카드 이동 개선

- 홈 화면의 상품 카드를 클릭하면 상품 목록으로 이동하던 동작을 해당 상품 상세 페이지로 바로 이동하도록 변경했다.
- `productId`를 상품 상세 라우트 인자로 전달한다.

## 5. 예약/주문 진행 상태 배지

예약 흐름 페이지 상단에 현재 단계 배지를 추가했다.

| 화면 | 표시 문구 |
|---|---|
| 예약함 | 예약 흐름 1/5 · 담아둔 상품 |
| 예약 전 확인 | 예약 흐름 2/5 · 예약 확인 |
| 주문서 작성 | 예약 흐름 3/5 · 주문서 작성 |
| 결제 | 예약 흐름 4/5 · 결제 대기 |
| 주문 완료 | 예약 흐름 5/5 · 주문 완료 |
| 주문 상세 | 배송 흐름 · 현재 주문 상태 |
| 반품 신청 | 반품 흐름 1/2 · 반품 신청 |
| 반품 완료 | 반품 흐름 2/2 · 접수 완료 |

모바일에서는 문구가 길어지지 않도록 상태명 중심으로 표시되게 구성했다.

## 6. 마이페이지 보강

- 마이페이지 상단에 사용자 요약 카드를 추가했다.
- 표시 정보:
  - 홍길동 님
  - minji.kim@harvest-slot.test
  - 진행 중 주문 2건
  - 최근 주문 상태 배송 완료
  - 기본 배송지 서울 강남구
- `최근 주문 확인` 버튼을 추가하고 더미 주문 상세 화면으로 이동하게 했다.
- 마이페이지 하단의 중복 기본 배송지 패널은 제거했다.
- 마이페이지 메뉴는 아래 구조로 정리했다.
  - 내 주문
  - 기본 배송지 관리
  - 회원 정보
  - 로그아웃

## 7. 배송지 관리 화면 추가

마이페이지의 `기본 배송지 관리` 버튼을 별도 배송지 관리 페이지로 연결했다.

추가된 화면:

- 배송지 관리 페이지
- 배송지 추가 페이지
- 배송지 수정 페이지

배송지 관리 페이지 구성:

- `배송지 추가하기` 버튼
- 기본 배송지 카드
- `기본 배송지`, `최근 사용` 배지
- 수정 버튼

배송지 추가/수정 페이지 구성:

- 이름
- 휴대폰번호
- 우편번호
- 주소 찾기
- 주소
- 상세주소
- 배송 요청사항
- 기본 배송지 설정 체크박스
- 저장하기 버튼

현재 `주소 찾기`는 카카오 주소 API 연결 예정 안내 팝업으로 처리했다.

## 8. 배송지 추가/수정 구분

- 배송지 추가 버튼 클릭 시 제목을 `배송지 추가`로 표시한다.
- 기존 배송지 수정 버튼 클릭 시 제목을 `배송지 수정`으로 표시한다.
- 수정 화면에는 기존 더미 배송지 정보를 미리 채워 표시한다.
- 저장 완료 문구도 추가/수정에 따라 다르게 표시한다.

## 9. 배송지 폼 검증

배송지 저장 시 아래 항목을 검증한다.

- 이름이 비어 있으면 `이름을 입력해주세요.`
- 휴대폰번호가 비어 있으면 `휴대폰번호를 입력해주세요.`
- 주소가 비어 있으면 `주소를 입력해주세요.`
- 상세주소가 비어 있으면 `상세주소를 입력해주세요.`

휴대폰번호 형식 검증은 아직 적용하지 않았다. 백엔드 연결 또는 기획 확정 후 강화하는 것이 좋다.

## 10. 배송 요청사항 직접 입력

- 배송 요청사항 드롭다운에 `직접 입력` 옵션을 추가했다.
- `직접 입력` 선택 시 요청사항 입력칸이 표시된다.
- 직접 입력을 선택했는데 입력값이 비어 있으면 저장을 막고 `배송 요청사항을 입력해주세요.` 안내를 표시한다.

## 11. 회원 정보 페이지 추가

마이페이지의 `회원 정보 보기` 버튼을 별도 회원 정보 페이지로 연결했다.

회원 정보 페이지 구성:

- 기본 정보 카드
  - 이름
  - 이메일
  - 휴대폰번호
- 이메일 줄에 `인증 완료` 배지를 표시한다.
- 계정 상태 카드는 제거하고 기본 정보 안에서 인증 상태를 가볍게 보여주는 방식으로 정리했다.
- 정보 수정 안내 카드
- 정보 수정 버튼

## 12. 회원 정보 수정 페이지 추가

회원 정보 페이지의 `정보 수정` 버튼을 별도 수정 페이지로 연결했다.

회원 정보 수정 페이지 구성:

- 이름 입력
- 이메일 입력
- 이메일 `인증하기` 버튼
- 휴대폰번호 입력
- 저장하기 버튼

현재 동작:

- 이메일 인증은 선택 사항으로 안내한다.
- 인증하지 않아도 저장 가능하다는 문구를 표시한다.
- `인증하기`는 실제 인증 메일 발송 없이 안내 팝업으로 처리한다.
- 저장 시 필수 입력값을 확인하고 `회원 정보가 저장되었습니다.` 안내 후 이전 화면으로 돌아간다.

## 14. 결제/주문 흐름 최신화

결제 화면과 결제 이후 주문 흐름을 다시 정리했다.

### 결제 화면

- 기존의 단순 결제 화면을 `결제 전 확인` 화면으로 재구성했다.
- 수확 예약 상품 영역을 추가했다.
- 배송 정보 영역을 추가했다.
- 카드 간편결제 선택 영역을 추가했다.
- 상품 금액, 배송비, 최종 결제 금액을 분리해서 표시했다.
- `수확 일정과 결제 금액을 확인했습니다.` 체크박스를 추가했다.
- 체크하지 않으면 결제 진행을 막고 안내 팝업을 표시한다.
- 수확 일정 변동 가능성과 농가 최종 확인 안내를 추가했다.

### 주문 완료 화면

- 결제 후 바로 배송 완료처럼 보이지 않도록 `농가 확인 중` 흐름으로 문구를 정리했다.
- 결제 완료 후 농가가 수확 가능 수량을 최종 확인한다는 안내를 추가했다.
- 주문 상세에서 선별과 배송 상태를 확인할 수 있다는 설명을 추가했다.

### 주문 상세 화면

- 결제 직후 주문 상태를 `농가 확인 중`으로 표시하도록 더미 데이터를 통일했다.
- 진행 현황 문구를 정상 한글로 정리했다.
- 주문 요약의 배송지, 결제 금액, 총 중량, 택배사, 송장번호를 통일했다.
- 반품 신청은 `배송 완료` 상태일 때만 가능하도록 화면 분기를 정리했다.
