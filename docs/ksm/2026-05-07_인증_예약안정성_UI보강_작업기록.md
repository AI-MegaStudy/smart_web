# 2026-05-07 인증, 예약 안정성, 주문 상태 UI 보강 작업 기록

작성일: 2026-05-07

이 문서는 2026-05-07 기준으로 사용자 웹에서 진행한 인증/노출 제어, 예약 가능 여부 예외 처리, 주문 상태 문구 정리, 주문 상세 배송 정보 표시, 예약 확인 UI 보강 내용을 정리한 문서다.

## 1. 이번 작업 목적

이번 작업의 목적은 아래 4가지였다.

- 로그인 상태와 비로그인 상태에 따라 노출을 더 자연스럽게 정리한다.
- 비로그인 사용자가 예약 흐름에 진입할 때 안내 후 로그인으로 유도한다.
- 예약 확인 단계에서 현재 주문 가능한지 한 번 더 확인하는 예외 처리를 추가한다.
- 주문 상태 문구와 주문 상세 배송 정보를 고객이 이해하기 쉽게 정리한다.

## 2. 완료한 작업

### 2-1. 인증과 노출 제어

완료 내용:

- 홈 헤더에서 로그인 상태일 때만 `마이페이지`가 보이도록 정리했다.
- 로그인 상태일 때 헤더에 `로그아웃` 버튼이 보이도록 분리했다.
- 비로그인 상태에서는 `회원가입`, `로그인`만 보이도록 정리했다.
- 마이페이지 내부의 기존 로그아웃 버튼은 유지했다.
- 비로그인 상태에서 상품 상세의 `예약함에 담기`를 누르면 로그인 필요 안내 후 로그인 화면으로 이동하도록 처리했다.
- 비로그인 상태에서 예약함의 `예약 진행하기`를 누르면 로그인 필요 안내 후 로그인 화면으로 이동하도록 처리했다.
- 마이페이지 직접 접근 시 로그인 상태가 아니면 로그인 화면으로 보내는 1차 방어를 추가했다.

적용 파일:

- `lib/app/router.dart`
- `lib/presentation/views/home/home_page.dart`
- `lib/presentation/views/products/product_detail_page.dart`
- `lib/presentation/views/basket/local_basket_page.dart`

### 2-2. 예약 가능 여부 예외 처리

완료 내용:

- 예약 확인 페이지에서 장바구니 항목을 다시 검증하도록 구성했다.
- 현재는 실제 백엔드 대신 더미 슬롯 데이터를 기준으로 아래 항목을 확인한다.
  - 선택한 슬롯이 아직 존재하는지
  - 슬롯 상태가 `OPEN`인지
  - 현재 남은 박스 수로 주문 가능한지
- 문제가 있으면 예약 확인 페이지 진입 시 팝업으로 안내한다.
- 문제가 있는 항목 카드에는 `재확인 필요` 상태 배지를 표시한다.
- 항목 안에 경고 안내 박스를 추가했다.
- 문제가 있을 경우 `주문서 작성하기` 버튼을 비활성화했다.
- 대신 `예약함으로 돌아가기` 버튼으로 수량을 다시 조정할 수 있게 했다.

현재 동작 의미:

- 아직 웹소켓이나 실시간 동기화는 아니고, 주문 직전 한 번 더 확인하는 mock 검증 단계다.
- 실제 정합성 보장은 나중에 백엔드 예약 생성 API가 담당해야 한다.

적용 파일:

- `lib/presentation/view_models/reservation_confirm_view_model.dart`
- `lib/presentation/views/reservation/reservation_confirm_page.dart`
- `lib/presentation/widgets/notice_box.dart`

### 2-3. 주문 진행 상태 문구 정리

완료 내용:

- 주문 상세 타임라인의 `선별 완료` 문구를 `발송 준비`로 변경했다.
- 설명 문구도 아래처럼 고객용 흐름에 맞게 정리했다.
  - 기존: `사과 신선도 선별 후 출고 준비가 완료됩니다.`
  - 변경: `농가 확인이 끝난 상품을 선별하고 발송을 준비하고 있습니다.`
- `배송 중` 설명 문구도 아래처럼 정리했다.
  - 기존: `선별을 마친 사과가 배송지로 이동 중입니다.`
  - 변경: `발송이 시작된 사과가 배송지로 이동 중입니다.`
- 주문 완료 화면 문구도 `선별과 배송 상태`에서 `발송 준비와 배송 상태`로 맞췄다.

적용 파일:

- `lib/presentation/view_models/order_detail_view_model.dart`
- `lib/presentation/views/orders/order_detail_page.dart`
- `lib/presentation/views/orders/order_complete_page.dart`

### 2-4. 주문 상세의 택배사, 송장번호 표시 방식 정리

완료 내용:

- 주문 상세의 `주문 요약` 레이아웃은 그대로 유지했다.
- `택배사`, `송장번호`는 주문 상태에 따라 다르게 표시되도록 정리했다.

표시 기준:

- `배송 중`, `배송 완료`, `반품 요청 접수`
  - 실제 택배사와 송장번호 표시
- 그 이전 상태
  - `배송 준비 중`
  - `송장번호 준비 중`

적용 파일:

- `lib/presentation/views/orders/order_detail_page.dart`

### 2-5. 예약 확인 페이지의 예약 내용 요약 카드 강조

완료 내용:

- 예약 확인 페이지 오른쪽 요약 카드의 배경을 따뜻한 크림 톤으로 변경했다.
- 테두리와 그림자를 추가해 시선이 먼저 가게 조정했다.
- 상단에 `예약 요약` 배지를 추가했다.
- 제목 색도 조금 더 또렷하게 정리했다.

현재 상태:

- 시각적으로는 더 잘 보이지만, 문구 자체는 사용자에게 약간 겹쳐 보일 수 있다는 피드백이 있었다.
- 특히 아래 조합이 다소 어색할 수 있다.
  - `예약 요약`
  - `예약 내용 요약`

적용 파일:

- `lib/presentation/views/reservation/reservation_confirm_page.dart`

## 3. 현재 확인된 문구 개선 포인트

예약 확인 페이지 요약 카드에서 아래 표현은 다시 다듬는 것이 좋다.

- `예약 요약`
- `예약 내용 요약`

현재 판단:

- 두 표현이 사용자 입장에서 비슷하게 들릴 수 있다.
- 배지를 제거하거나, 제목을 더 자연스럽게 바꾸는 쪽이 좋다.

추천 대안:

- 제목만 `예약 상품 확인`으로 변경
- 또는
  - 배지: `주문 전 확인`
  - 제목: `예약 상품 확인`

개인적으로는 아래 조합이 가장 단순하고 자연스럽다.

```text
배지 제거
제목: 예약 상품 확인
```

## 4. 현재 확인된 안내 카드 강조 개선 포인트

예약 확인 페이지 요약 카드 안의 아래 두 안내 박스를 더 잘 보이게 만들 수 있다.

- `농가가 정한 수확 예정 기간입니다. ...`
- `다음 단계에서 배송 정보를 입력하고 ...`

추천 방향:

### 첫 번째 안내 카드

- 의미: 수확 일정 변동 가능성 안내
- 추천 톤: 연한 웜 옐로 또는 크림
- 추천 아이콘: 일정, 캘린더 계열

### 두 번째 안내 카드

- 의미: 다음 단계 안내
- 추천 톤: 연한 그린 또는 민트
- 추천 아이콘: 확인, 다음 단계 계열

추천 조합:

- 첫 번째 안내 카드: 일정 안내 톤
- 두 번째 안내 카드: 다음 단계 안내 톤
- 전체 카드 강조는 지금보다 약간만 유지
