# 사용자 웹 DB 수정 없는 개선 우선순위

이 문서는 Harvest Slot 사용자 웹에서 DB 구조를 추가하거나 수정하지 않고, 기존 기획 범위 안에서 화면 완성도를 높일 수 있는 개선 작업을 정리한 문서입니다.

참고 사이트:

- `https://www.yggudapp.kr/`
- `https://www.yggudapp.kr/31`
- `https://www.yggudapp.kr/35`
- `https://www.yggudapp.kr/36`
- `https://www.yggudapp.kr/50`

주의사항:

- 참고 사이트의 문구를 그대로 복사하지 않는다.
- Harvest Slot 기획에 맞게 예약, 수확 일정, 잔여 수량, 선별, 배송, 반품 안내 중심으로 재작성한다.
- 게시판, 댓글, 검색, 글쓰기, 조회수, 관리자 콘텐츠 등록 기능은 DB가 필요하므로 이번 범위에서 제외한다.
- 외부 패키지가 필요한 기능을 추가하려면 먼저 pub.dev에서 패키지를 확인하고 사용자 허락을 받은 뒤 적용한다.

## 1순위. 홈 메인 히어로 이미지 슬라이더

목표:

- 홈 첫 화면의 완성도를 높인다.
- 과수원/사과 이미지를 크게 보여주고, Harvest Slot의 핵심 가치를 바로 전달한다.

구현 방식:

- Flutter 기본 `PageView` 사용
- `PageController`로 좌우 이동
- 좌우 화살표 버튼 추가
- 하단 dot indicator 추가
- 선택 사항: `Timer`로 4~5초마다 자동 전환

DB 필요 여부:

- 고정 이미지 3~4장과 고정 문구로 구현하면 DB 수정 없음

추천 문구:

```text
수확일이 확정된 사과만 예약하세요
농가가 직접 확정한 수확 예정 범위와 잔여 수량을 확인합니다.

수확 후 선별하여 순차 배송
예약한 수확 슬롯에 맞춰 신선한 사과를 받아보세요.

기상과 생육 상황까지 안내
수확 일정 변동 가능성을 미리 확인하고 안심하고 예약합니다.
```

## 2순위. 홈 스크롤 등장 애니메이션

목표:

- 사용자가 스크롤을 내릴 때 섹션이 자연스럽게 나타나는 느낌을 준다.
- 정적인 홈 화면을 더 풍성하게 만든다.

구현 방식:

- 외부 패키지 없이 `AnimatedOpacity`, `AnimatedSlide`, `ScrollController` 조합으로 구현 가능
- 섹션이 화면 근처에 들어오면 opacity와 offset을 변경한다.

DB 필요 여부:

- DB 수정 없음

주의사항:

- 너무 과한 애니메이션은 피한다.
- 예약 웹이므로 정보 확인을 방해하지 않는 정도로만 적용한다.
