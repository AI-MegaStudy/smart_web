# SMART_WEB 발표용 데모 ID 확정안
작성일: 2026-05-12

## 1. 목적

이 문서는 발표 영상과 리허설에 사용할
`SMART_WEB` 데모용 상품, 예약, 주문 ID를 현재 DB 기준으로 확정한 문서다.

## 2. 확정 값

현재 발표용 기본값은 아래와 같이 사용한다.

- 대표 상품 ID: `3`
- 주문서 작성용 예약 ID: `62`
- 결제/주문 완료용 주문 ID: `30`
- 배송 완료/반품 신청용 주문 ID: `26`

## 3. 선정 이유

### 3.1 대표 상품 ID `3`

- 현재 발표 흐름에서 대표 상품 진입용으로 이미 가장 안정적으로 쓰고 있다.
- 상품 상세와 수확 슬롯 흐름 연결이 자연스럽다.

### 3.2 예약 ID `62`

- 최근 예약 데이터이며 `RESERVED` 상태로 확인됐다.
- 주문서 작성 화면 진입용 예약 기준값으로 쓰기 적절하다.

### 3.3 결제/주문 완료용 주문 ID `30`

- 주문번호: `ORD-OWNER3-REQUESTED`
- 주문 상태: `PROCUREMENT_REQUESTED`

선정 이유:

- 결제 완료 이후 바로 배송 완료로 보이는 것보다
  농가 확인 중 흐름으로 이어지는 것이 발표상 더 자연스럽다.
- 주문 완료 화면에서 `결제 완료 후 다음 단계가 진행된다`는 설명과 잘 맞는다.

### 3.4 배송 완료/반품 신청용 주문 ID `26`

- 주문번호: `ORD-202605306`
- 주문 상태: `DELIVERED`
- 배송 상태: `DELIVERED`

선정 이유:

- 주문 상세에서 `배송 완료` 상태를 안정적으로 보여줄 수 있다.
- 반품 신청 버튼 노출 조건과 맞는다.
- 반품 신청 화면 진입용 주문 기준값으로 적합하다.

## 5. 발표 흐름 적용 방식

현재 발표용 코치 투어는 아래처럼 나눠서 쓴다.

```text
홈 -> 상품 목록 -> 상품 상세 -> 예약함 -> 예약 확인
-> 주문서 작성(예약 ID 62)
-> 결제(주문 ID 30)
-> 주문 완료(주문 ID 30)
-> 주문 상세(주문 ID 26)
-> 반품 신청(주문 ID 26)
```

이렇게 분리한 이유:

- 주문서와 결제 단계는 진행 중 주문 흐름이 더 자연스럽다.
- 주문 상세와 반품 신청은 배송 완료 주문이 필요하다.
