# 팀장 컨펌 반영 작업 체크리스트

작성일: 2026-05-07

이 문서는 팀장 컨펌 기준으로 사용자 웹에서 순차 반영해야 하는 작업을 정리한 문서다.

정리 기준은 아래와 같다.

- 우선순위
- 화면 또는 영역
- 작업 내용
- 백엔드 확인 필요 여부
- 비고

## 2. 백엔드 확인이 필요한 항목

- 예약 가능 여부 확인 방식
- 관리자 또는 점주 잔여 박스 수량과 사용자 웹 수량 불일치 시 응답 구조
- 배송지 목록 조회, 선택, 기본값 반영 방식
- 주문 진행 상태 문구와 실제 서버 상태 매핑
- 배송중 상태에서 택배사 노출용 필드
- 로그인 자동 유지 시 토큰, 쿠키 저장 정책
- 카카오 주소검색 패키지 적용 시점

## 3. 우선 작업 묶음 제안

### 1차. 인증과 노출 제어

- 로그인 시에만 `마이페이지` 노출
- 로그아웃 버튼 추가
- 비로그인 상태 예약 차단 및 로그인 안내
- 자동 로그인 정책 검토

### 2차. 예약과 주문 안정성

- 예약 가능 여부 체크
- 잔여 박스 불일치 예외 처리
- 주문 진행 상태 문구 정리
- 배송중 상태 택배사 표시

### 3차. 배송과 결제 UX

- 배송지 목록에서 선택
- 선택 시 자동 채움
- 배송지 별명 필드 추가
- 결제 수단 더미 UI
- 결제 준비중 팝업

### 4차. 메인 화면 개선

- 수확일 배너 스타일 수정
- 품종 필터 추가
- 섹션 구분선, 여백 정리
- 품목 소개 가독성 개선
- 이번주 오픈 수확 슬롯 노출 정책 정리

### 5차. 폼과 검증 보강

- 회원가입 정규화
- 회원가입 팝업 UX 교체
- 반품 상세 사유 글자 수 검증
- 부분 금액 옵션 제거

### 6차. 후순위 보강

- 플로팅 버튼
- 카테고리 점프 이동
- 모바일 고객센터 전화 연결
- 카카오 주소검색 연결

## 4. 메모

- 실제 구현은 `AGENTS.md`와 `docs/ksm/사용자웹_필요정보_정리.md` 기준을 우선한다.
- 고객 화면에는 내부 상태 코드, API 경로, DB 명칭을 직접 노출하지 않는다.
- 상태 문구는 반드시 고객용 한글 문구로 정리한다.
- 예약과 주문 가능 여부는 프론트 단독 판단보다 서버 검증 결과를 우선 반영한다.

## 5. 2026-05-08 반영 현황

오늘 확인 또는 반영한 항목은 아래와 같다.

### 완료 또는 확인 완료

- 로그인 시에만 `마이페이지` 노출 확인
- 로그인 상태에서 `로그아웃` 버튼 노출 확인
- 비로그인 상태 예약 진입 시 로그인 안내 확인
- 예약 가능 여부 사전 확인 로직 확인
- 잔여 박스 불일치 시 팝업 안내와 주문서 작성 비활성화 확인
- 배송지 목록 선택 UI 추가
- 기존 배송지 선택 시 입력 필드 자동 채움 추가
- 배송지 별명 필드 추가
  - 단, 별명은 선택 입력값으로 처리
- 더미 카드결제 수단 유지
- 결제 수단 드롭다운 UI 추가
- 준비중 결제 수단 선택 시 팝업 표시 추가
- 예약 확인 요약 카드 안내 문구 역할별 강조
- 메인 패럴랙스 섹션의 버튼형 배지를 캡션 라인형으로 변경
- `이번주 오픈 수확 슬롯`은 대표/최신 3개 노출 정책으로 확인

### 백엔드 전달 필요

- 홈 대표 상품 API는 아래 형태를 권장한다.

```text
GET /products?featured=true&limit=3
```

- 주문서 작성 화면의 배송지 선택을 위해 배송지 목록 API 응답에 배송지 별명, 기본 배송지 여부, 최근 사용 여부가 있으면 좋다.

```text
address_id
address_label
receiver_name
receiver_phone
zip_code
address
detail_address
delivery_memo
is_default
is_recent
```

## 6. 2026-05-08 추가 반영 현황

오늘 추가로 반영하거나 확인한 항목은 아래와 같다.

### 완료

- 메인 화면의 품종 소개, 보관 안내, 즐기는 방법, 예약 흐름, 자주 묻는 질문 영역을 카드형 섹션으로 정리했다.
- 품종 카드의 모바일 잘림과 과한 세로 여백을 줄였다.
- 메인 화면 빠른 이동 메뉴를 추가했다.
  - 위로
  - 상품
  - 품종
  - 보관
  - 흐름
  - FAQ
- 빠른 이동 메뉴는 모바일 하단 중앙, 데스크톱 오른쪽 중간 위치에 표시되도록 했다.
- 모바일에서 빠른 이동 메뉴가 푸터를 가리지 않도록 하단 여백을 보정했다.
- 회원가입 화면에 이메일, 비밀번호, 이름, 전화번호 실시간 검증 안내를 추가했다.
- 회원가입 조건이 맞지 않으면 가입 버튼이 비활성화되도록 했다.
- 반품 신청 화면에서 부분 반품 금액 선택을 제거했다.
- 반품 상세 사유는 10자 이상 입력해야 신청할 수 있도록 했다.
- 주문 상세의 반품 신청 버튼은 배송 완료 상태에서만 보이는 기준으로 유지했다.

### 백엔드 전달 또는 추후 확인

- 메인 화면의 이번 주 오픈 수확 슬롯은 프론트에서는 최신 3개 노출 기준으로 유지한다.
- 백엔드에는 아래 형태를 권장한다.

```text
GET /products?featured=true&limit=3
```

- 상품 목록의 품종 필터는 이미 별도 화면에 있으므로 홈에 중복 추가하지 않는다.
- 배송지 목록 API에는 배송지 별명, 기본 배송지 여부, 최근 사용 여부가 있으면 좋다.

```text
address_label
is_default
is_recent
```

### 남은 확인

- 실제 브라우저 모바일 폭에서 빠른 이동 메뉴 위치 최종 확인
- 주요 화면별 모바일 overflow 최종 점검
- 카카오 주소 검색은 패키지 검토와 사용자 승인 후 별도 진행

## 7. 2026-05-08 모바일 최종 점검 반영

주요 사용자 흐름 화면을 기준으로 모바일 레이아웃 위험 요소를 다시 확인했다.

확인 범위:

- 상품 목록
- 상품 상세
- 예약함
- 예약 확인
- 주문서 작성
- 결제
- 내 주문
- 주문 상세
- 반품 신청
- 반품 요청 완료

추가 반영:

- 반품 신청 화면의 반품 사유 선택 영역은 모바일에서 고정 높이 그리드 대신 세로 카드 목록으로 표시되도록 정리했다.
- 결제 화면의 요약 행은 오른쪽 값이 길어져도 화면 밖으로 밀리지 않도록 유연한 텍스트 영역으로 보정했다.

현재 완료로 볼 수 있는 항목:

- 로그인 상태별 상단바 분기
- 마이페이지 접근 수단 상단 표시
- 로고 클릭 시 홈 이동
- 비로그인 예약 진입 시 로그인 안내
- 예약 확인 전 수량/슬롯 상태 재검증
- 배송지 선택 카드와 자동 채움
- 결제 수단 드롭다운과 준비중 팝업
- 예약 확인 안내 카드 강조
- 홈 안내 배너 캡션형 정리
- 홈 최신 3개 수확 슬롯 노출 기준 정리
- 홈 섹션 카드화
- 홈 빠른 이동 메뉴
- 회원가입 실시간 검증
- 반품 사유 10자 이상 검증
- 반품 부분 금액 선택 제거
- 주요 화면 모바일 overflow 1차 점검

남은 항목:

- 실제 브라우저에서 모바일 크기별 최종 눈검수
- 실제 API 연결 시 상태 문구와 응답 필드 최종 매핑
- 카카오 주소 검색은 팀장 추천 패키지 `kpostal_plus`로 프론트 연결을 시작함
- 고객센터 전화번호 클릭 연결 여부 결정

## 8. 카카오 주소 검색 적용 현황

팀장이 사용 경험이 있는 `kpostal_plus` 패키지를 우선 적용했다.

적용 범위:

- 배송지 추가 화면
- 배송지 수정 화면
- 주문서 작성 화면
- `주소 찾기` 버튼
- `주소 검색` 버튼

현재 동작:

- `주소 찾기` 버튼을 누르면 `KpostalPlusView` 주소 검색 화면으로 이동한다.

## 10. 2026-05-08 사과 카테고리 정리

팀장 피드백에 따라 사용자 웹의 사과 카테고리를 두 가지로 정리했다.

반영 기준:

- 기존 더미 품종 `후지`, `홍로`, `시나노골드`, `감홍` 제거
- 사용자 화면의 사과 카테고리는 `양광`, `부사`만 사용
- 상품 목록 필터도 더미 상품의 `variety` 기준으로 `양광`, `부사`만 노출
- 홈 품종 소개 카드도 `양광`, `부사` 두 개로 축소
- 예약함 더미 상품명도 `양광 사과`, `부사 사과`로 맞춤

백엔드 전달 필요:

- 상품 API의 `variety_name` 또는 카테고리 응답값은 MVP 기준 `양광`, `부사` 중심으로 맞추는 것이 좋다.

## 11. 2026-05-08 고객센터/푸터 연결

후순위 보강 항목이었던 고객센터 영역을 프론트 기준으로 연결했다.

반영 내용:

- 홈 하단 고객센터 카드의 전화번호를 클릭 가능하게 변경
- 전화번호 클릭 시 `tel:` 링크로 전화 앱을 열도록 처리
- 전화 앱을 열 수 없는 환경에서는 고객용 안내 팝업 표시
- 푸터에 `이메일 문의` 링크 추가
- 이메일 문의 클릭 시 `mailto:` 링크로 메일 앱을 열도록 처리

사용 패키지:

```text
url_launcher: ^6.3.2
```

비고:

- PC 브라우저에서는 전화 앱이 연결되어 있지 않으면 전화 연결이 제한될 수 있다.
- 고객센터 전화번호와 이메일은 실제 운영 정보 확정 시 교체하면 된다.
