# 팀장 컨펌 반영 작업 체크리스트

작성일: 2026-05-07

이 문서는 팀장 컨펌 기준으로 사용자 웹에서 순차 반영해야 하는 작업을 정리한 문서다.

정리 기준은 아래와 같다.

- 우선순위
- 화면 또는 영역
- 작업 내용
- 백엔드 확인 필요 여부
- 비고

## 1. 작업 체크리스트

| 우선순위 | 화면/영역 | 작업 내용 | 백엔드 확인 | 비고 |
|---|---|---|---|---|
| 1 | 공통 헤더 / 메인 | 로그인 시에만 `마이페이지` 노출 | 아니오 | 현재 로그인 상태 분기 점검 필요 |
| 1 | 공통 헤더 / 메인 | 로그아웃 버튼 추가 | 아니오 | 마이페이지 내부 외에 메인 접근성도 고려 |
| 1 | 예약 진입 | 비로그인 상태에서 예약 시 로그인 필요 안내 | 아니오 | 상품 상세, 예약 버튼 흐름 모두 확인 필요 |
| 1 | 예약 페이지 | 예약 가능 여부 사전 확인 로직 정리 | 예 | 점주 페이지 데이터와 잔여 박스 불일치 대응 |
| 1 | 예약 페이지 | 잔여 박스 불일치 시 팝업 안내 후 선택 불가 처리 | 예 | 웹소켓 대신 현재는 FastAPI 예외 처리 우선 |
| 1 | 배송 정보 입력 | 배송지 목록에서 선택 기능 추가 | 예 | 실제 API 연동 시 주소 목록 필요 |
| 1 | 배송 정보 입력 | 기존 배송지 선택 시 입력 필드 자동 채움 | 예 | 기본 배송지, 선택 배송지 반영 구조 필요 |
| 1 | 배송 정보 입력 | 배송지 별명 필드 추가 | 예 | 예시: `내집`, `회사` |
| 1 | 결제 수단 | 더미 카드결제 수단 추가 | 아니오 | 실제 결제 연동 전 준비중 UX |
| 1 | 결제 수단 | 결제 수단 드롭다운 UI 구성 | 아니오 | `준비중` 항목 포함 가능 |
| 1 | 결제 수단 | 결제 버튼 클릭 시 `준비중` 팝업 표시 | 아니오 | 실제 승인 대신 안내 처리 |
| 2 | 예약 내용 요약 | 예약 내용 요약 카드 또는 영역 색상 강조 | 아니오 | 시선이 먼저 가도록 톤 조정 |
| 2 | 메인 페이지 | 수확일 예약 배너를 버튼처럼 안 보이게 수정 | 아니오 | 정보 배너, 태그 느낌으로 변경 |
| 2 | 메인 페이지 | `예약 가능한 상품` 품종 필터링 추가 | 아니오 | 현재 더미 데이터 기준 먼저 가능 |
| 2 | 메인 페이지 | 설명 정보와 상품 영역 사이 구분선 추가 | 아니오 | 섹션 분리감 강화 |
| 2 | 메인 페이지 | 설명 정보와 상품 영역 여백 확대 | 아니오 | 모바일 포함 점검 |
| 2 | 메인 페이지 | 품목 소개를 더 크게, 한눈에 들어오게 개선 | 아니오 | 카드, 타이포, 레이아웃 재조정 |
| 2 | 메인 페이지 | `이번주 오픈 수확 슬롯` 노출 방식 정리 | 아니오 | 최신 3개 제한 또는 품종 필터 중 택1 |
| 2 | 주문 진행 상태 | `선별완료` 문구를 `발주완료` 맥락 기준으로 재정리 | 예 | 백엔드 상태명과 화면 문구 매핑 점검 필요 |
| 2 | 주문 진행 상태 | `배송중` 상태에 택배사 정보 표시 | 예 | 배송 API 응답 필드 확인 필요 |
| 2 | 내 주문 / 반품 | 반품 상세 사유 최소 글자 수 검증 추가 | 아니오 | 예: 10자 이상 등 정책 결정 필요 |
| 2 | 내 주문 / 반품 | 요청 금액 선택에서 `부분` 옵션 제거 | 아니오 | 전체 반품 중심 UX로 정리 |
| 3 | 회원가입 | hidden text 처리 정리 | 아니오 | 의미 확인 필요, 스팸 방지용인지 점검 |
| 3 | 회원가입 | 이름 입력 검증 강화 | 아니오 | 최소 길이, 공백 처리 등 |
| 3 | 회원가입 | 전화번호 정규화 | 아니오 | 하이픈 포함, 제거 정책 통일 |
| 3 | 회원가입 | 이메일 정규화 | 아니오 | trim, lowercase 등 |
| 3 | 회원가입 | 기존 팝업 안내를 다른 UX로 교체 | 아니오 | 인라인 메시지 또는 배너 방식 권장 |
| 3 | 로그인 | 자동 로그인 쿠키 저장 기능 추가 | 예 | 실제 토큰 저장 정책 확인 필요 |
| 3 | 공통 UI | 페이지 중간, 상단 이동 플로팅 버튼 추가 | 아니오 | 긴 페이지 우선 적용 |
| 3 | 공통 UI | 카테고리 선택 시 해당 구역으로 점프 이동 | 아니오 | 메인, 상품 목록 우선 검토 |
| 4 | 예약 내용 요약 / 배송지 | 카카오 주소검색 패키지 연결 | 예 | 택권님 패키지 추가 이후 진행 |
| 4 | 고객센터 모바일 | 전화번호 탭 시 바로 전화 연결 | 아니오 | 시간 남을 때 진행 |

## 2. 백엔드 확인이 필요한 항목

- 예약 가능 여부 확인 방식
- 관리자 또는 점주 잔여 박스 수량과 사용자 웹 수량 불일치 시 응답 구조
- 배송지 목록 조회, 선택, 기본값 반영 방식
- 주문 진행 상태 문구와 실제 서버 상태 매핑
- 배송중 상태에서 택배사 노출용 필드
- 로그인 자동 유지 시 토큰, 쿠키 저장 정책
- 카카오 주소검색 패키지 적용 시점

## 3. 우선 작업 묶음 제안

### 1차. 인증과 노출 제어

- 로그인 시에만 `마이페이지` 노출
- 로그아웃 버튼 추가
- 비로그인 상태 예약 차단 및 로그인 안내
- 자동 로그인 정책 검토

### 2차. 예약과 주문 안정성

- 예약 가능 여부 체크
- 잔여 박스 불일치 예외 처리
- 주문 진행 상태 문구 정리
- 배송중 상태 택배사 표시

### 3차. 배송과 결제 UX

- 배송지 목록에서 선택
- 선택 시 자동 채움
- 배송지 별명 필드 추가
- 결제 수단 더미 UI
- 결제 준비중 팝업

### 4차. 메인 화면 개선

- 수확일 배너 스타일 수정
- 품종 필터 추가
- 섹션 구분선, 여백 정리
- 품목 소개 가독성 개선
- 이번주 오픈 수확 슬롯 노출 정책 정리

### 5차. 폼과 검증 보강

- 회원가입 정규화
- 회원가입 팝업 UX 교체
- 반품 상세 사유 글자 수 검증
- 부분 금액 옵션 제거

### 6차. 후순위 보강

- 플로팅 버튼
- 카테고리 점프 이동
- 모바일 고객센터 전화 연결
- 카카오 주소검색 연결

## 4. 메모

- 실제 구현은 `AGENTS.md`와 `docs/ksm/사용자웹_필요정보_정리.md` 기준을 우선한다.
- 고객 화면에는 내부 상태 코드, API 경로, DB 명칭을 직접 노출하지 않는다.
- 상태 문구는 반드시 고객용 한글 문구로 정리한다.
- 예약과 주문 가능 여부는 프론트 단독 판단보다 서버 검증 결과를 우선 반영한다.
