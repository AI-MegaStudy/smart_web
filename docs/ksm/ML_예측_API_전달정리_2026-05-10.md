# ML 예측 API 전달 정리

작성일: 2026-05-10

## 1. 전달받은 자료

백엔드 담당자로부터 `아카이브 3.zip`과 `harvest-slot-backend` 레포 기준 ML 예측 API 정보를 전달받았다.

전달 파일 구성:

```text
model.joblib
predict_server.py
inference.py
requirements.txt
```

`predict_server.py`는 독립 실행용 FastAPI 예시이며, SMART_WEB 백엔드에 그대로 넣는 파일은 아니다. SMART_WEB에서는 해당 로직을 기존 `backend/app/services/ml_service.py` 구조에 맞춰 반영한다.

## 2. API 요약

OWNER 로그인 후 받은 access token으로 호출한다.

```text
POST /api/v1/owner/ml/predictions
Authorization: Bearer {access_token}
```

요청값:

```json
{
  "farm_id": 1,
  "product_id": 1,
  "features": {
    "past_yield_kg": 3000,
    "market_price": 5000,
    "variety": "부사",
    "mar_avg_temp": 8.5,
    "aug_sunshine": 210.0,
    "oct_rainfall": 65.0,
    "aug_humidity": 72.0
  }
}
```

응답 주요 필드:

```text
prediction_id
farm_id
product_id
unit_yield_kg_10a
estimated_yield_kg
suggested_reservable_min_kg
suggested_reservable_max_kg
recommended_price
predicted_harvest_start
predicted_harvest_end
confidence
safety_factor
warning_message
model_version
```

## 3. SMART_WEB 반영 내용

반영 파일:

```text
backend/app/ml_models/model.joblib
backend/app/services/ml_service.py
backend/app/schemas/ml_schema.py
backend/requirements.txt
```

반영 기준:

- `backend/app/ml_models/model.joblib`에서 모델을 lazy loading 방식으로 1회 로드한다.
- 모델 입력 피처 순서는 `mar_avg_temp`, `aug_sunshine`, `oct_rainfall`, `aug_humidity`로 고정한다.
- `MLPredictionCreateRequest.features`에 검증 범위를 추가했다.
- 예측 결과는 기존 `ml_predictions` 테이블에 저장한다.
- 응답 구조는 기존 공통 구조인 `{ data, message, error }`를 유지한다.

## 4. 고객 웹 적용 기준

이 API는 OWNER 전용이다.

SMART_WEB 고객 예약 웹에서는 ML 예측값을 직접 노출하지 않는다. 고객 화면에서는 백엔드/농가가 확정한 상품, 수확 슬롯, 예약 가능 정보만 표시한다.

고객 웹에서 직접 표시하지 않는 값:

```text
unit_yield_kg_10a
estimated_yield_kg
confidence
safety_factor
model_version
```

## 5. 실행 환경 메모

백엔드 담당자 전달 기준:

```text
Python 3.12
scikit-learn 1.8.0
```

다만 zip 내부 `requirements.txt`에는 `scikit-learn==1.4.2`가 적혀 있었고, `harvest-slot-backend`의 `requirements.txt`에는 `scikit-learn==1.8.0`이 적혀 있었다. SMART_WEB에서는 백엔드 담당자 메시지와 `harvest-slot-backend` 기준에 맞춰 `scikit-learn==1.8.0`을 우선 기록한다.

실행 전 설치 예시:

```text
pip install -r backend/requirements.txt
```

서버 실행 예시:

```text
uvicorn backend.app.main:app --reload
```

Swagger 확인:

```text
http://localhost:8000/docs
```
