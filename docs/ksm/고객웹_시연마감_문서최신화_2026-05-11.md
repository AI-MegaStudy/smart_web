# 고객웹 시연 마감 문서 최신화

작성일: 2026-05-11

## 1. 현재 고객웹 기준

2026-05-11 시연 마감 점검 기준으로 고객웹은 아래 상태를 최신 기준으로 본다.

- 고객 화면 품종 기준은 `부사 사과`, `양광 사과`이다.
- 고객 화면 농장명은 `문경 햇살 농장` 하나로 고정한다.
- 과거 문서의 `신고 사과`, `신고/부사` 기준은 당시 작업 기록이며 현재 화면 기준이 아니다.
- 고객 화면에는 `QA`, `더미`, `DB`, `API`, `Mock`, `ML`, `테스트 코드` 같은 개발자용 표현을 직접 노출하지 않는다.
- 내부 코드명 `MockPaymentPage`, `MockAuthSession`, API 경로 `mock-approve`는 고객 화면 문구가 아니라 구현/백엔드 연결 이름이다.
- 고객 화면에는 ML 예측값, 모델 버전, 예측 근거를 직접 노출하지 않는다.

## 2. 시연 전 최종 반영 사항

- 홈 품종 소개와 홈 배지를 `양광` 기준으로 다시 변경했다.
- 홈, 상품, 예약함, 주문 상세의 농장명 표시를 `문경 햇살 농장`으로 통일했다.
- 로컬 더미 상품명과 상품 API 매핑에서 `양광 사과`가 표시되도록 정리했다.
- 주문 상품명 보정 로직에서 기존 `신고 배` 계열 응답도 `양광 사과`로 보정한다.
- 결제 대기 주문의 결제 정보에서는 승인금액 `0원` 대신 결제 예정 금액을 표시한다.
- 반품 요청 화면에서 증빙 이미지를 여러 장 선택할 수 있게 했다.
- 반품 요청 화면에서 요청 금액을 전체 금액 또는 직접 입력으로 선택할 수 있게 했다.
- 반품 요청 요약에는 증빙 이미지 선택 수와 요청 금액이 즉시 반영된다.
- 푸터의 이용약관, 개인정보처리방침은 고객센터 안내 대신 스크롤 가능한 본문 모달로 표시한다.

## 3. 더미 데이터 기준

고객 웹 QA 더미 데이터는 실제 고객처럼 보이도록 정리한다.

- 고객명 예시: 김민지, 박서준, 이지은
- 상품명 예시: 충주 햇살 부사 사과, 충주 햇살 양광 사과
- 농장명 예시: 문경 햇살 농장
- 주소 예시: 서울시 강남구 테헤란로 152 9층
- 화면 노출값에는 `QA 고객`, `Harvest Demo Farm`, `customer@test.com` 같은 시연용 값이 보이지 않아야 한다.

## 4. 반품 증빙 이미지 주의

현재 백엔드는 `return_requests.evidence_image_url` 문자열 필드 하나만 받는다.

프론트에서는 여러 장을 선택할 수 있게 했고, 서버에는 `/mock/returns/파일명` 형식의 경로를 쉼표로 묶어 전달한다.

실제 파일 업로드 API가 추가되면 현재 파일 선택 UI는 유지하고, 서버 전달값만 업로드 결과 URL 목록으로 교체하면 된다.
