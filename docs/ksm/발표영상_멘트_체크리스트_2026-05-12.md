# SMART_WEB 발표영상 멘트 체크리스트
작성일: 2026-05-12

## 1. 문서 목적

이 문서는 `SMART_WEB` 발표 영상을 찍을 때
팀장이 화면 흐름에 맞춰 자연스럽게 설명할 수 있도록
짧은 발표 멘트를 정리한 체크리스트다.

원칙:

- 한 화면에 한 메시지만 말한다.
- 개발 용어보다 고객 관점 표현을 우선한다.
- `Mock`, `QA`, `DB`, `API` 같은 표현은 말하지 않는다.
- 멘트는 짧고 끊어서 말한다.

## 2. 시작 멘트

```text
SMART_WEB은 고객이 농가의 수확 일정에 맞춰 상품을 예약하고,
주문과 결제, 배송, 반품까지 한 번에 확인할 수 있는 Flutter Web 서비스입니다.
```

짧게 줄이면:

```text
고객이 수확 슬롯 기준으로 예약부터 주문, 배송, 반품까지 진행하는 사용자 웹입니다.
```

## 3. 화면별 멘트

### 3.1 회원가입 / 로그인

```text
처음 이용하는 고객은 회원가입과 이메일 인증 후 로그인해서 서비스를 이용할 수 있습니다.
오늘 시연은 로그인된 상태에서 핵심 예약 흐름부터 보겠습니다.
```

### 3.2 홈

```text
홈에서는 서비스 소개와 함께 상품, 품종, 보관, FAQ 같은 주요 정보를 빠르게 확인할 수 있습니다.
```

### 3.3 홈 빠른 이동 메뉴

```text
이 메뉴를 통해 홈 안의 주요 섹션으로 빠르게 이동하면서 전체 서비스를 짧게 소개할 수 있습니다.
```

### 3.4 상품 목록

```text
고객은 예약 가능한 상품과 수확 예정 범위를 확인하고 원하는 상품을 선택할 수 있습니다.
```

### 3.5 상품 상세

```text
상품 상세에서는 농가 정보와 상품 정보를 보고, 원하는 수확 슬롯을 선택할 수 있습니다.
```

### 3.6 예약함

```text
선택한 상품은 예약함에 담아서 수량과 금액을 한 번 더 확인할 수 있습니다.
```

### 3.7 예약 확인

```text
예약 확인 단계에서는 최종 예약 수량과 예상 금액을 점검한 뒤 주문서 작성으로 넘어갑니다.
```

### 3.8 주문서 작성

```text
주문서에서는 수령인과 배송지를 입력하고 실제 결제 전에 주문 내용을 다시 확인합니다.
```

### 3.9 결제

```text
결제 화면에서는 최종 결제 금액과 결제 수단을 확인한 뒤 주문을 완료합니다.
```

### 3.10 주문 완료

```text
결제가 끝나면 주문 번호와 현재 주문 상태를 바로 확인할 수 있습니다.
```

### 3.11 주문 상세

```text
주문 상세에서는 결제 이후 농가 확인, 배송 진행, 배송 완료까지의 흐름을 한 화면에서 볼 수 있습니다.
```

### 3.12 반품 신청

```text
배송 완료 후 문제가 있는 경우에는 반품 사유와 증빙 이미지를 첨부해 웹에서 직접 반품을 신청할 수 있습니다.
```

## 4. 마무리 멘트

```text
이렇게 SMART_WEB은 상품 탐색부터 예약, 주문, 결제, 배송, 반품까지
고객의 전체 구매 여정을 하나의 웹 흐름으로 연결합니다.
```

짧게 줄이면:

```text
즉, 고객이 수확 예약형 상품을 처음부터 끝까지 스스로 진행할 수 있도록 만든 사용자 웹입니다.
```

## 5. 말할 때 주의할 점

- 한 화면에서 1문장 이상 길게 설명하지 않는다.
- 고객이 무엇을 할 수 있는지 중심으로 말한다.
- 내부 구현 방식 설명은 영상보다 발표 질의응답에서 다루는 편이 좋다.
- 반품은 반드시 `배송 완료 후 가능`이라고 설명한다.

## 6. 권장 진행 방식

아래 리듬으로 말하면 안정적이다.

```text
화면 이름
-> 고객이 여기서 무엇을 하는지
-> 다음 화면으로 왜 넘어가는지
```

예시:

```text
여기는 상품 목록 화면입니다.
고객은 예약 가능한 상품과 수확 예정 범위를 확인할 수 있습니다.
이제 상품 상세로 들어가서 수확 슬롯을 선택해보겠습니다.
```
