# 커밋/푸시 요약 - ML 예측 API

작성일: 2026-05-10

## 커밋 메시지 후보

```text
feat: OWNER ML 예측 모델 연동 기반 추가
```

## 변경 요약

- `backend/app/ml_models/model.joblib`을 배치했다.
- `backend/app/services/ml_service.py`에서 Mock 계산식을 제거하고 `model.joblib` 기반 예측으로 변경했다.
- `backend/app/schemas/ml_schema.py`에 ML 요청값 검증 범위를 추가했다.
- `backend/requirements.txt`에 ML 실행 의존성(`scikit-learn`, `pandas`, `numpy`, `joblib`)을 포함했다.
- `docs/ksm/ML_예측_API_전달정리_2026-05-10.md`에 전달받은 ML API 적용 기준을 정리했다.
- `AGENTS.md`에 ML 예측 API 전달 정리 문서와 고객 웹 노출 제한 기준을 추가했다.

## 적용 기준

- ML API는 OWNER 전용 `POST /api/v1/owner/ml/predictions`로 유지한다.
- 고객 예약 웹에는 ML 예측값, 신뢰도, 모델 버전, 내부 예측 근거를 직접 표시하지 않는다.
- 고객 웹은 백엔드/농가가 확정한 상품, 수확 슬롯, 예약 가능 정보만 표시한다.

## 검증 결과

문법 검증:

```powershell
python -m py_compile backend\app\services\ml_service.py backend\app\schemas\ml_schema.py backend\app\routers\ml_router.py
```

모델 로딩 및 샘플 예측 검증:

```powershell
python -c "from backend.app.services.ml_service import get_ml_model, build_model_input; model=get_ml_model(); df=build_model_input({'mar_avg_temp':8.5,'aug_sunshine':210.0,'oct_rainfall':65.0,'aug_humidity':72.0}); print(round(float(model.predict(df)[0]), 2))"
```

결과:

```text
1509.54
```

주의:

- 현재 로컬 Python 환경에서는 scikit-learn `1.7.2`로 모델을 로드해 `InconsistentVersionWarning`이 출력되었다.
- 모델 저장 버전은 `1.8.0` 기준이므로 실제 백엔드 실행 환경은 `backend/requirements.txt`의 `scikit-learn==1.8.0`에 맞추는 것이 안전하다.
