# SMART_WEB 발표영상 촬영 전 데모 ID 체크리스트
작성일: 2026-05-12

## 1. 문서 목적

이 문서는 `SMART_WEB` 발표 영상을 찍기 전에
데모 흐름이 끊기지 않도록 발표용 상품, 예약, 주문 기준값과 사전 점검 항목을 정리한 문서다.

중요한 점:

- 지금 단계는 영상 촬영 단계가 아니다.
- 지금은 촬영 전에 화면 흐름을 안정화하는 준비 단계다.
- 실제 촬영은 이 문서의 점검이 끝난 뒤 진행한다.

## 2. 현재 발표용 데모 기본값

현재 코치 투어 기본값은 아래 파일에서 관리한다.

- [customer_demo_route_defaults.dart](C:/Users/tj/Documents/GitHub/smart_web/lib/demo/customer_demo_route_defaults.dart:1)

현재 설정값:

```dart
featuredProductId = 3
reservationId = 5
deliveredOrderId = 8
```

의미:

- `featuredProductId`
  - 홈에서 대표 상품으로 진입할 때 사용할 상품 ID
- `reservationId`
  - 예약 확인 다음 주문서 작성 화면으로 진입할 때 사용할 예약 ID
- `deliveredOrderId`
  - 결제, 주문 완료, 주문 상세, 반품 신청 흐름에 사용할 주문 ID

## 3. 현재 기준 해석

### 3.1 대표 상품

현재 `productId = 3`은 발표 흐름상 유지해도 괜찮다.

근거:

- 로컬 상품 기준에서 `productId = 3`은 대표 상품 역할로 쓰기 적절하다.
- 품종 기준도 현재 발표 문서와 맞는 `양광/부사` 흐름 안에 있다.

### 3.2 예약 ID와 주문 ID

`reservationId = 5`, `deliveredOrderId = 8`은 지금은 발표용 임시 기본값으로 보면 된다.

즉:

- 코드 연결용 기본값으로는 쓸 수 있다.
- 실제 촬영 전에는 발표 계정 기준 실데이터로 다시 확인하는 것이 안전하다.

특히 `deliveredOrderId`는 아래 조건이 꼭 맞아야 한다.

- 주문 상세 진입 가능
- 배송 상태가 `배송 완료`
- 반품 신청 버튼 노출 가능
- 반품 신청 화면 진입 가능

## 4. 촬영 전 꼭 확인할 항목

아래 항목이 끝나야 영상 촬영을 시작하는 게 맞다.

### 4.1 상품 확인

- `featuredProductId`가 실제로 존재하는가
- 상품 상세 화면이 정상 진입되는가
- 수확 슬롯이 비어 있지 않은가
- 상품 이미지가 기본 이미지가 아니라 실제 이미지로 보이는가

### 4.2 예약 흐름 확인

- 예약함에 상품을 담을 수 있는가
- 예약 확인 화면으로 정상 이동하는가
- `reservationId` 기준 주문서 작성 화면이 비어 있지 않은가

### 4.3 주문 흐름 확인

- 주문서 작성 화면에서 수령인, 연락처, 주소가 정상 보이는가
- 결제 화면으로 이동 가능한가
- 주문 완료 화면이 정상 노출되는가

### 4.4 주문 상세와 반품 확인

- `deliveredOrderId` 주문이 실제로 열리는가
- 주문 상태가 `배송 완료`로 보이는가
- 주문 상세에서 `반품 신청` 버튼이 보이는가
- 반품 신청 화면 진입이 되는가

## 5. 촬영 직전 최종 체크

촬영 직전에는 아래 순서로 1회 리허설하는 것이 좋다.

```text
로그인
-> 홈 3탭 숨김 트리거
-> 홈
-> 상품 목록
-> 상품 상세
-> 예약함
-> 예약 확인
-> 주문서 작성
-> 결제
-> 주문 완료
-> 주문 상세
-> 반품 신청
```

체크 기준:

- 중간에 빈 화면이 없어야 한다.
- 버튼 비활성 상태가 나오지 않아야 한다.
- 발표 멘트와 화면 흐름이 어긋나지 않아야 한다.
- 반품 신청 직전 주문 상태가 반드시 `배송 완료`여야 한다.

## 6. 촬영은 언제 하나

촬영은 아래 두 조건이 끝난 뒤 하는 것이 맞다.

1. 발표용 상품/주문 ID 확정
2. 리허설 1회 이상 성공

즉, 순서는 아래처럼 본다.

```text
데모 ID 정리
-> 화면 흐름 점검
-> 리허설
-> 최종 멘트 점검
-> 영상 촬영
```
