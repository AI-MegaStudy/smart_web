# SMART_WEB 발표영상 촬영 순서표

## 1. 문서 목적

이 문서는 발표 영상을 실제로 찍을 때
어떤 순서로 화면을 보여주고 어떤 지점에서 멈추면 좋은지 정리한 촬영용 순서표다.

이 문서는 설명용이 아니라 촬영 진행용이다.

## 2. 촬영 전 준비

촬영 시작 전 아래 항목을 먼저 끝낸다.

- 로그인 상태 확인
- 발표용 데모 ID 확인
- 홈 3탭 숨김 트리거 동작 확인
- 상품 상세, 주문 상세, 반품 신청 화면 진입 가능 여부 확인

## 3. 촬영 흐름

### 3.1 오프닝

- 홈 진입
- 2~3초 정지
- 브랜드와 메인 비주얼 노출

권장 멘트:

```text
SMART_WEB은 고객이 수확 일정에 맞춰 상품을 예약하고 주문과 배송까지 확인할 수 있는 사용자 웹입니다.
```

### 3.2 홈 소개

- 홈 3탭 숨김 트리거 실행
- 코치 투어 시작
- 우측 빠른 이동 메뉴 노출
- 2초 정지

권장 멘트:

```text
홈에서는 상품과 품종, 보관 정보, FAQ까지 주요 정보를 빠르게 확인할 수 있습니다.
```

### 3.3 상품 목록

- 코치 투어 다음 단계
- 첫 상품 카드 노출
- 2초 정지

권장 멘트:

```text
고객은 예약 가능한 상품과 수확 예정 범위를 보고 원하는 상품을 선택합니다.
```

### 3.4 상품 상세

- 상품 상세 진입
- 대표 CTA와 수확 슬롯 영역 중심으로 노출
- 2~3초 정지

권장 멘트:

```text
상품 상세에서는 농가 정보와 상품 내용을 확인하고 수확 슬롯을 선택할 수 있습니다.
```

### 3.5 예약함

- 예약함 진입
- 담긴 상품과 CTA 노출
- 2초 정지

권장 멘트:

```text
선택한 상품은 예약함에서 수량과 금액을 다시 확인할 수 있습니다.
```

### 3.6 예약 확인

- 예약 확인 진입
- 우측 요약 영역 중심으로 노출
- 2초 정지

권장 멘트:

```text
예약 확인 단계에서는 최종 예약 수량과 예상 금액을 점검합니다.
```

### 3.7 주문서 작성

- 주문서 작성 진입
- 배송지와 결제 전 요약 노출
- 2초 정지

권장 멘트:

```text
주문서에서는 수령인과 배송지를 입력하고 주문 내용을 다시 확인합니다.
```

### 3.8 결제

- 결제 화면 진입
- 결제 버튼과 금액 요약 노출
- 2초 정지

권장 멘트:

```text
결제 단계에서는 최종 결제 금액과 결제 수단을 확인한 뒤 주문을 완료합니다.
```

### 3.9 주문 완료

- 주문 완료 화면 진입
- 주문 번호와 상태 노출
- 2초 정지

권장 멘트:

```text
결제가 끝나면 주문 번호와 현재 상태를 바로 확인할 수 있습니다.
```

### 3.10 주문 상세

- 주문 상세 진입
- 진행 타임라인과 배송 상태 노출
- 3초 정지

권장 멘트:

```text
주문 상세에서는 결제 이후 농가 확인과 배송 진행 상태를 한 화면에서 볼 수 있습니다.
```

### 3.11 반품 신청

- 반품 신청 진입
- 반품 사유, 증빙 이미지, 반품 요청 버튼 노출
- 3초 정지

권장 멘트:

```text
배송 완료 후 문제가 있는 경우에는 웹에서 직접 반품 요청도 진행할 수 있습니다.
```

### 3.12 마무리

- 반품 신청 화면 또는 주문 상세 화면에서 종료
- 2초 정지

권장 멘트:

```text
이처럼 SMART_WEB은 상품 탐색부터 예약, 주문, 결제, 배송, 반품까지 고객의 전체 흐름을 연결합니다.
```

## 4. 촬영 팁

- 각 화면은 최소 2초 이상 머문다.
- 주문 상세와 반품 신청은 3초 정도 더 길게 보여준다.
- 화면 전환 직후 바로 말하기보다 0.5초 정도 쉬고 말하면 안정적이다.
- 마우스 이동은 천천히 한다.
- 스크롤은 짧고 분명하게 한다.

## 5. NG 체크

- 빈 화면이 나오면 다시 찍는다.
- 반품 신청 버튼이 안 보이면 다시 점검 후 촬영한다.
- 주문 상태가 `배송 완료`가 아니면 반품 파트는 촬영하지 않는다.
- 화면 문구에 개발자용 표현이 보이면 먼저 수정하고 촬영한다.
